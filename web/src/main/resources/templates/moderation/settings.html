Jackpot

Jackpot pool tuning. The pool grows from loss tributes and trade fees and pays out on a small chance per casino-game win. Use the eligibility gates below (RTP cap, winner cooldown, activity window) to keep the pool from being swept by high-RTP games or low-activity edge cases. Set JACKPOT_WIN_PCT to 0 to disable jackpot wins entirely; set JACKPOT_RTP_MAX_PCT to a value below a game's canonical RTP to disable jackpots just for that game.

%

Each segment is a wedge of the spinning wheel that's spun when a player wins the jackpot. Weight is how often it's picked (positive integer; relative, not %). Payout % is how much of the pool the winner takes when the wheel lands there. Default 80:1,10:5,5:10,4:20,1:50 — most wins give a small pity prize, a rare 1-in-100 spin pays out half the pool. For a flat fixed-percent payout, set a single segment (e.g. 1:30 for "always 30% of the pool").

Anti-autoclicker

Suspected autoclickers (same screen pixel clicked repeatedly with no cursor motion between bets) get a per-(user, guild) "streak" that ramps the house edge by 2.5 percentage points per consecutive bot-like click, capped per game by the values below. The cap is the forced-loss probability applied on top of the fair game; the suspect's effective RTP becomes (1 − cap/100) × fair RTP. Legitimate play never trips the detector. Set 0 to disable a game's gate entirely. Discord call paths are unaffected (no mouse signal).
